## 🕐 时间排序规范 ⭐️ 新增 (v2.3)

### 排序规则

**所有股票数据必须按 `last_updated` 字段降序排序**（最新的在前）

#### 1. **股票列表排序**

在 `stocks_master.json` 或分片文件中，股票数组应按 `last_updated` 降序排列：

```json
{
  "stocks": [
    {
      "name": "股票 A",
      "code": "000001",
      "last_updated": "2026-05-06",  // 最新更新的在前
      ...
    },
    {
      "name": "股票 B",
      "code": "000002",
      "last_updated": "2026-05-05",
      ...
    }
  ]
}
```

#### 2. **文章列表排序**

每只股票的 `articles` 数组应按 `date` 降序排列（最新的文章在前）：

```json
{
  "articles": [
    {
      "title": "最新文章标题",
      "date": "2026-05-06",  // 最新的文章在前
      "source": "https://...",
      ...
    },
    {
      "title": "较旧的文章",
      "date": "2026-05-05",
      ...
    }
  ]
}
```

#### 3. **排序实现代码**

**Python 示例**：
```python
# 股票列表按 last_updated 降序排序
stocks.sort(key=lambda x: x.get('last_updated', ''), reverse=True)

# 文章列表按 date 降序排序
for stock in stocks:
    if 'articles' in stock:
        stock['articles'].sort(key=lambda x: x.get('date', ''), reverse=True)
```

**JavaScript 示例**：
```javascript
// 股票列表按 last_updated 降序排序
stocks.sort((a, b) => (b.last_updated || '').localeCompare(a.last_updated || ''));

// 文章列表按 date 降序排序
stocks.forEach(stock => {
    if (stock.articles) {
        stock.articles.sort((a, b) => (b.date || '').localeCompare(a.date || ''));
    }
});
```

#### 4. **索引文件排序**

`stocks_index.json` 中的股票也应按 `last_updated` 降序排列：

```json
{
  "version": "2.0",
  "last_updated": "2026-05-06",
  "total_stocks": 100,
  "stocks": {
    "000001": {
      "name": "股票 A",
      "last_updated": "2026-05-06",  // 最新
      "file": "2026-05-06.json"
    },
    "000002": {
      "name": "股票 B",
      "last_updated": "2026-05-05",
      "file": "2026-05-05.json"
    }
  }
}
```

## 🏗️ 第一层：股票基本信息

### 完整结构

```json
{
  "stocks": [
    {
      "name": "股票名称",
      "code": "股票代码",
      "board": "交易所",
      "industry": "行业分类",
      "concepts": ["概念标签"],
      "products": ["产品"],
      "core_business": ["核心业务"],
      "industry_position": ["行业地位"],
      "chain": ["产业链位置"],
      "partners": ["合作伙伴"],
      "mention_count": 提及次数,
      "valuation": {
        "target_market_cap": "目标市值",
        "target_market_cap_billion": 800.0
      },
      "articles": [
        {
          "title": "文章标题",
          "date": "文章日期",
          "source": "文章链接",
          "accidents": ["催化剂/事件"],
          "insights": ["投资洞察"],
          "key_metrics": ["关键指标"],
          "target_valuation": ["目标估值"]
        }
      ]
    }
  ]
}
```

### 字段详细说明

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|--------|------|------|------|------|
| **name** | string | ✅ | 股票名称 | `"金风科技"` |
| **code** | string | ✅ | 股票代码 | `"002202"` |
| **board** | string | ✅ | 交易所 | `"SZ"` 或 `"SH"` |
| **industry** | string | ✅ | 行业分类（三级路径） | `"电力设备 - 风电设备 - 风电整机"` |
| **concepts** | array | ✅ | 概念标签数组 | `["风电", "储能", "碳中和"]` |
| **products** | array | ⏳ | 公司产品（待补充） | `["风机整机"]` |
| **core_business** | array | ⏳ | 核心业务（待补充） | `["风电整机制造"]` |
| **industry_position** | array | ⏳ | 行业地位（待补充） | `["全球第一"]` |
| **chain** | array | ⏳ | 产业链位置（待补充） | `["下游 - 整机"]` |
| **partners** | array | ⏳ | 合作伙伴（待补充） | `["华为", "腾讯"]` 或 `[]` |
| **mention_count** | number | ⏳ | 提及次数（待统计） | `20` |
| **valuation** | object | ⏳ | 估值信息（待补充） | `{"target_market_cap": "800 亿", "target_market_cap_billion": 800.0}` |
| **articles** | array | ✅ | 文章列表（至少 1 篇） | 见第二层结构 |

### 字段规范

#### 1. **industry** - 行业分类
- **格式**: 三级路径，用 `-` 连接
- **示例**: `"电力设备 - 风电设备 - 风电整机"`
- **来源**: 同花顺行业映射表

#### 2. **concepts** - 概念标签
- **格式**: 字符串数组
- **来源**: 所属概念映射表（按分号分割）
- **示例**:
  ```json
  "concepts": [
    "风电",
    "储能",
    "碳中和",
    "绿色电力",
    "深股通"
  ]
  ```

#### 3. **products** - 公司产品
- **格式**: 字符串数组
- **内容**: 公司生产的主要产品、设备、材料等
- **提取规则**:
  - 从 raw material 的「产品」或「业务亮点」部分提取
  - 提取具体产品名称（如：涂布模头、辊压设备、铝塑膜等）
  - 如无明确产品信息，使用空数组 `[]`

**示例**:
```json
"products": [
  "湿法涂布模头",
  "干法电极整线设备",
  "混料设备",
  "纤维化设备",
  "多辊成膜设备"
]
```

#### 4. **core_business** - 核心业务
- **格式**: 字符串数组
- **内容**: 公司的主营业务、核心收入来源
- **提取规则**:
  - 从 raw material 的「公司概况」或「业务亮点」部分提取
  - 描述公司的主要业务领域和商业模式
  - 如无明确信息，使用空数组 `[]`

**示例**:
```json
"core_business": [
  "物料自动化处理产线及设备的研发、生产和销售",
  "粉料、粒料、液料、浆料等散装物料处理",
  "锂电池、精细化工等行业自动化解决方案"
]
```

#### 5. **industry_position** - 行业地位
- **格式**: 字符串数组
- **内容**: 公司在行业中的地位、排名、竞争优势
- **提取规则**:
  - 从 raw material 提取市占率、排名、技术领先等信息
  - 包括：市占率第一、龙头、领先、唯一等描述
  - 如无明确信息，使用空数组 `[]`

**示例**:
```json
"industry_position": [
  "湿法涂布模头市占率领先",
  "辊压设备龙头",
  "叠片设备全球市占率第一",
  "国内低轨卫星电源市场占有率 50%-60%"
]
```

#### 6. **chain** - 产业链位置
- **格式**: 字符串数组，使用标准产业链分层
- **内容**: 公司在产业链中的位置（上游/中游/下游）
- **标准分层**:
  - **上游**: 原材料、核心零部件、基础材料
  - **中游**: 设备制造、材料加工、模组组装
  - **下游**: 终端应用、系统集成、运营服务
- **提取规则**:
  - 根据行业特性判断产业链位置
  - 格式：`["上游 - 原材料", "中游 - 设备"]` 或 `["下游 - 整机"]`

**示例**:
```json
// 设备厂商
"chain": [
  "中游 - 锂电设备",
  "中游 - 固态电池设备"
]

// 材料厂商
"chain": [
  "上游 - 电池材料",
  "上游 - 固态电解质"
]

// 整机厂商
"chain": [
  "下游 - 风电整机",
  "下游 - 储能系统"
]
```

#### 7. **partners** - 合作伙伴/客户
- **格式**: 字符串数组
- **内容**: 主要客户、合作伙伴、供应商
- **提取规则**:
  - 从 raw material 的「客户资源」「合作伙伴」部分提取
  - 包括：宁德时代、比亚迪、华为、富士康等
  - 如无明确信息，使用空数组 `[]`

**示例**:
```json
"partners": [
  "宁德时代",
  "比亚迪",
  "亿纬锂能",
  "清研电子",
  "华为"
]
```

#### 8. **valuation** - 估值信息（股票级别）⭐️ 新增
- **格式**: JSON 对象
- **内容**: 目标市值、目标价、PE 等估值数据
- **提取规则**:
  - 从 raw material 的「估值数据」「目标市值」部分提取
  - 支持多种估值指标
  - **新增**: `target_market_cap_billion` 字段用于数值计算和排序

**结构**:
```json
"valuation": {
  "target_market_cap": "目标市值（如：800 亿）",
  "target_market_cap_billion": 800.0,
  "target_price": "目标价（如：66.56 元）",
  "current_market_cap": "当前市值（如：251 亿）",
  "current_price": "当前股价",
  "pe": "PE 估值（如：2026 年 52 倍 PE）",
  "upside": "上涨空间（如：80.18%）",
  "rating": "评级（如：买入、增持）"
}
```

**示例**:
```json
"valuation": {
  "target_market_cap": "800 亿",
  "target_market_cap_billion": 800.0,
  "target_price": "66.56 元",
  "pe": "2026 年 52 倍 PE",
  "upside": "80.18%",
  "rating": "买入"
}
```

**字段说明**:
- `target_market_cap`: 目标市值的文本表示（如："800 亿"、"150e"）
- `target_market_cap_billion`: 目标市值的数值表示（**单位：亿元**），用于排序和计算
  - 示例："800 亿" → 800.0
  - 示例："150e" → 150.0
  - 示例："1.5 千亿" → 1500.0
  - 示例："5000 万" → 0.5

## 📄 第二层：文章详情

### 完整结构

```json
{
  "articles": [
    {
      "title": "文章标题",
      "date": "文章日期",
      "source": "文章链接",
      "accidents": ["催化剂/事件"],
      "insights": ["投资洞察"],
      "key_metrics": ["关键指标"],
      "target_valuation": ["目标估值"]
    }
  ]
}
```

### 字段详细说明

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|--------|------|------|------|------|
| **title** | string | ⏳ | 文章标题（待补充） | `"风电出海零关税政策催化"` |
| **date** | string | ✅ | 文章日期（YYYY-MM-DD） | `"2026-03-13"` |
| **source** | string | ✅ | 文章链接（微信公众号 URL） | `"https://mp.weixin.qq.com/s/..."` |
| **accidents** | array | ✅ | 催化剂/事件（与文章对应） | `["回购 + 绿色甲醇催化"]` |
| **insights** | array | ✅ | 投资洞察（与文章对应） | `["受益于风电出海零关税政策"]` |
| **key_metrics** | array | ✅ | 关键指标（与文章对应） | `["海外最强"]` |
| **target_valuation** | array | ✅ | 目标估值（与文章对应） | `["目标市值", "空间"]` |
| **expected_price** | array | ⏳ | 预期价格/目标价 | `["2026 年：84.87 元", "2027 年：101.85 元"]` |
| **expected_performance** | array | ⏳ | 预计业绩/盈利预测 | `["2025 年扣非净利润：8.893 亿元", "2026-2027 年盈利增速：20%"]` |
| **market_valuation** | array | ⏳ | 市场估值/估值模型 | `["成长股估值模型", "估值公式：[8.5 + 2 X 增长率] X EPS"]` |

### 字段规范

#### 1. **accidents** - 催化剂/事件
- **格式**: 字符串数组
- **长度**: 建议 ≤60 字，超过需优化
- **内容**: 简短的事件描述，不含分析推理
- **优化规则**:
  - 清理序号（`1、`、`2、`）
  - 清理格式标记（`#标签#`、`【标题】`）
  - 提取核心信息（中标、涨价、订单等）
  - 超过 60 字截取前 40 字 + `...`

**示例对比**:
```json
// ❌ 优化前（过长）
"accidents": ["公司在过去 20 天内分别上调拉丁美洲、中东、非洲、土耳其、东南亚、南亚地区 MDI 或 TDI 价格 200-300 美元/吨，匈牙利 BC 也宣布对所有 MDI 产品涨价 300 欧元/吨"]

// ✅ 优化后
"accidents": ["MDI/TDI 海外多地区涨价 200-300 美元/吨"]
```

#### 2. **insights** - 投资洞察
- **格式**: 字符串数组
- **内容**: 投资逻辑、核心观点、深度分析
- **长度**: 不限制，保持原文

**示例**:
```json
"insights": [
  "🔥美股商业航天电源 Vicor 大涨，星载电源逻辑加强 0106，关注新雷能",
  "公司在 800G 光模块领域占据全球 50% 以上市场份额，且 1.6T 产品研发进度领先全球"
]
```

#### 3. **key_metrics** - 关键指标
- **格式**: 字符串数组
- **内容**: 市场地位、财务数据、产能等量化指标

**示例**:
```json
"key_metrics": [
  "市场地位与客户：公司是国内低轨卫星电源市场占有率最高的企业，市占率达 50%-60%",
  "毛利率超 42%，单瓦成本较国际竞品低 20% 以上"
]
```

#### 4. **target_valuation** - 目标估值
- **格式**: 字符串数组
- **内容**: 估值分析、目标价、市场空间等
- **当前**: 保持长文本格式

**示例**:
```json
"target_valuation": [
  "市场空间：按新雷能主流产品计算，标准低轨卫星电源系统单星价值量约 180 万元",
  "未来 5 年国内 5 万颗低轨卫星发射规划中，按 55% 市占率测算，对应市场空间达 495 亿元"
]
```

#### 5. **expected_price** - 预期价格/目标价
- **格式**: 字符串数组
- **内容**: 分析师给出的目标价、预期价格区间
- **来源**: 从文章中提取的估值测算

**示例**:
```json
"expected_price": [
  "2026 年：84.87 元",
  "2027 年：101.85 元"
]
```

#### 6. **expected_performance** - 预计业绩/盈利预测
- **格式**: 字符串数组
- **内容**: 盈利预测、业绩预期、增长率等
- **来源**: 从文章中提取的财务预测

**示例**:
```json
"expected_performance": [
  "2025 年扣非净利润：8.893 亿元",
  "2025 年扣非 EPS: 1.75 元/股",
  "2026-2027 年盈利增速：20%"
]
```

#### 7. **market_valuation** - 市场估值/估值模型
- **格式**: 字符串数组
- **内容**: 使用的估值模型、估值方法、估值逻辑
- **来源**: 从文章中提取的估值方法论

**示例**:
```json
"market_valuation": [
  "成长股估值模型",
  "估值公式：[8.5 + 2 X 增长率] X EPS"
]
```

## 📈 Dashboard 显示规范

### 个股页面估值显示

在个股详情页的估值板块中，增加 `target_market_cap_billion` 的显示：

```html
<!-- 估值信息 -->
{% if stock.valuation and (stock.valuation.target_market_cap or stock.valuation.target_market_cap_billion) %}
```

```python
def sort_stocks_by_update_time(stocks: List[Dict]) -> List[Dict]:
    """按 last_updated 降序排序"""
    return sorted(stocks, key=lambda x: x.get('last_updated', ''), reverse=True)
```

2. **每只股票的 articles 按 `date` 降序排序**
```python
def sort_articles_by_date(stock: Dict) -> Dict:
    """按 article date 降序排序"""
    if 'articles' in stock:
        stock['articles'].sort(
            key=lambda x: x.get('date', ''), 
            reverse=True
        )
    return stock
```

**完整排序流程**：
```python
def finalize_stock_data(stocks: List[Dict]) -> List[Dict]:
    """完成所有排序并返回最终数据"""
    # 1. 对每只股票的文章排序
    for stock in stocks:
        sort_articles_by_date(stock)
    
    # 2. 对股票列表排序
    sorted_stocks = sort_stocks_by_update_time(stocks)
    
    return sorted_stocks
```

```python
    "fields": {
        "name": {"stringValue": stock.get("name", "")},
        "code": {"stringValue": code},
        "last_updated": {"stringValue": stock.get("last_updated", "")},
        # ... 其他字段
    }
}

# 增量合并 articles
def merge_articles(existing_articles: list, new_articles: list) -> list:
    """按 (source, title) 去重合并"""
    existing_set = set()
    for article in existing_articles:
        existing_set.add((article.get('source', ''), article.get('title', '')))
    
    merged = existing_articles.copy()
    for article in new_articles:
        key = (article.get('source', ''), article.get('title', ''))
        if key not in existing_set:
            merged.append(article)
            existing_set.add(key)
    
    return merged
```
