# 股票数据存储架构

## 增量更新机制

### 文件结构

```
data/master/
├── stocks_master.json          # 主文件（完整数据，可选）
├── stocks_index.json           # 索引文件（股票代码列表和最后更新时间）
├── stocks/
│   ├── 2026-04-16.json        # 按日期分片存储
│   ├── 2026-04-15.json
│   ├── 2026-04-14.json
│   └── ...
└── README.md                   # 本文件
```

### 数据分片规则

1. **按日期分片**：每天更新的股票存储在单独的 JSON 文件中
2. **索引文件**：stocks_index.json 记录所有股票代码和最后更新时间
3. **增量加载**：前端优先加载最近 N 天的数据，需要时再加载历史数据

### stocks_index.json 格式

```json
{
  "version": "2.0",
  "last_updated": "2026-04-16",
  "total_stocks": 1500,
  "stocks": {
    "688227": {
      "name": "品高股份",
      "last_updated": "2026-04-16",
      "file": "2026-04-16.json"
    },
    "300593": {
      "name": "新雷能",
      "last_updated": "2026-04-16",
      "file": "2026-04-16.json"
    }
  }
}
```

### 每日分片文件格式

```json
{
  "date": "2026-04-16",
  "update_count": 15,
  "stocks": {
    "688227": { ... },
    "300593": { ... }
  }
}
```

### 加载策略

1. **首次加载**：读取 stocks_index.json，加载最近 7 天的数据
2. **滚动加载**：用户滚动到页面底部时，加载更早的数据
3. **搜索过滤**：在全部数据中搜索（需要时异步加载历史数据）

### 更新流程

1. 更新当天分片文件（如 2026-04-16.json）
2. 更新索引文件 stocks_index.json
3. 可选：更新主文件 stocks_master.json（完整备份）

### 优点

- 减少初始加载时间
- 支持按时间维度查询
- 便于数据归档和备份
- 支持并行加载多个分片
