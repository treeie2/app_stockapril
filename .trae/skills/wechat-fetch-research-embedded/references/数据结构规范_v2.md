## 🏗️ 第一层：股票基本信息

### 完整结构

```json
{
  "stocks": {
    "603115": {
      "name": "海星股份",
      "code": "603115",
      "board": "SH",
      "industry": "电子-电子化学品-电子化学品Ⅲ",
      "concepts": ["电极箔", "铝电解电容"],
      "products": ["电极箔"],
      "core_business": ["电极箔研发与生产"],
      "industry_position": ["国内电极箔龙头"],
      "chain": ["中游-电极箔制造"],
      "partners": [],
      "mention_count": 5,
      "articles": [
        {
          "title": "今天的一些信息整理5.12",
          "date": "2026-05-12",
          "source": "https://mp.weixin.qq.com/s/...",
          "accidents": ["电极箔涨价"],
          "insights": ["受益于铝电解电容需求增长"],
          "key_metrics": ["市占率30%"],
          "target_valuation": ["目标市值50亿"]
        }
      ],
      "last_updated": "2026-05-12"
    }
  }
}
```

> **注意**：顶层是 `stocks` 对象（keyed by code），**不是** `stocks` 数组。这是 v2.0+ 的规范变更。

### 字段详细说明

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|--------|------|------|------|------|
| **name** | string | ✅ | 股票名称 | `"海星股份"` |
| **code** | string | ✅ | 股票代码 | `"603115"` |
| **board** | string | ✅ | 交易所 | `"SZ"` 或 `"SH"` |
| **industry** | string | ✅ | 行业分类（三级路径） | `"电子-电子化学品-电子化学品Ⅲ"` |
| **concepts** | array | ✅ | 概念标签数组 | `["电极箔", "铝电解电容"]` |
| **products** | array | ⏳ | 公司产品 | `["电极箔"]` |
| **core_business** | array | ✅ | 核心业务 | `["电极箔研发与生产"]` |
| **industry_position** | array | ✅ | 行业地位 | `["国内电极箔龙头"]` |
| **chain** | array | ✅ | 产业链位置 | `["中游-电极箔制造"]` |
| **partners** | array | ✅ | 合作伙伴 | `["华为", "腾讯"]` 或 `[]` |
| **mention_count** | number | ⏳ | 提及次数 | `5` |
| **articles** | array | ✅ | 文章列表 | 见第二层结构 |
| **last_updated** | string | ✅ | 最后更新日期 | `"2026-05-12"` |

### 字段规范

#### 1. **industry** - 行业分类
- **格式**: 三级路径，用 `-` 连接（**无空格**）
- **示例**: `"电子-电子化学品-电子化学品Ⅲ"`
- **来源**: 同花顺行业映射表
- **禁止**: 使用"创业板/科创板"等板块名

#### 2. **concepts** - 概念标签
- **格式**: 字符串数组
- **来源**: 所属概念映射表
- **示例**:
  ```json
  "concepts": ["电极箔", "铝电解电容", "新能源"]
  ```

#### 3. **partners** - 合作伙伴
- **格式**: 字符串数组，如无数据则为空数组 `[]`
- **不使用 null**

## 📄 第二层：文章详情

### 完整结构

```json
{
  "articles": [
    {
      "title": "今天的一些信息整理5.12",
      "date": "2026-05-12",
      "source": "https://mp.weixin.qq.com/s/...",
      "accidents": ["电极箔涨价"],
      "insights": ["受益于铝电解电容需求增长"],
      "key_metrics": ["市占率30%"],
      "target_valuation": ["目标市值50亿"],
      "expected_price": ["2026年: 20元"],
      "expected_performance": ["2026年净利润: 2亿元"],
      "market_valuation": ["成长股估值模型"]
    }
  ]
}
```

### 字段详细说明

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|--------|------|------|------|------|
| **title** | string | ✅ | 文章标题 | `"今天的一些信息整理5.12"` |
| **date** | string | ✅ | 文章日期（YYYY-MM-DD） | `"2026-05-12"` |
| **source** | string | ✅ | 文章链接（微信公众号 URL） | `"https://mp.weixin.qq.com/s/..."` |
| **accidents** | array | ✅ | 催化剂/事件（≤60字/条） | `["电极箔涨价"]` |
| **insights** | array | ✅ | 投资洞察 | `["受益于铝电解电容需求增长"]` |
| **key_metrics** | array | ✅ | 关键指标 | `["市占率30%"]` |
| **target_valuation** | array | ✅ | 目标估值 | `["目标市值50亿"]` |
| **expected_price** | array | ⏳ | 预期价格/目标价 | `["2026年: 20元"]` |
| **expected_performance** | array | ⏳ | 预计业绩/盈利预测 | `["2026年净利润: 2亿元"]` |
| **market_valuation** | array | ⏳ | 市场估值/估值模型 | `["成长股估值模型"]` |

### 字段规范

#### 1. **accidents** - 催化剂/事件
- **格式**: 字符串数组
- **长度**: 建议 ≤60 字，超过需优化
- **内容**: 简短的事件描述，不含分析推理
- **优化规则**:
  - 清理序号（`1、`、`2、`）
  - 清理格式标记（`#标签#`、`【标题】`）
  - 提取核心信息（中标、涨价、订单等）
  - 超过 60 字截取前 40 字 + `...`

**示例对比**:
```json
// ❌ 优化前（过长）
"accidents": ["公司在过去 20 天内分别上调拉丁美洲、中东、非洲、土耳其、东南亚、南亚地区 MDI 或 TDI 价格 200-300 美元/吨"]

// ✅ 优化后
"accidents": ["MDI/TDI 海外多地区涨价 200-300 美元/吨"]
```

#### 2. **insights** - 投资洞察
- **格式**: 字符串数组
- **内容**: 投资逻辑、核心观点、深度分析
- **长度**: 不限制，保持原文

#### 3. **key_metrics** - 关键指标
- **格式**: 字符串数组
- **内容**: 市场地位、财务数据、产能等量化指标

#### 4. **target_valuation** - 目标估值
- **格式**: 字符串数组
- **内容**: 估值分析、目标价、市场空间等

#### 5. **expected_price** - 预期价格/目标价
- **格式**: 字符串数组
- **内容**: 分析师给出的目标价、预期价格区间

#### 6. **expected_performance** - 预计业绩/盈利预测
- **格式**: 字符串数组
- **内容**: 盈利预测、业绩预期、增长率等

#### 7. **market_valuation** - 市场估值/估值模型
- **格式**: 字符串数组
- **内容**: 使用的估值模型、估值方法、估值逻辑

## 🔧 数据处理流程

```
微信公众号文章 URL
    ↓
[Step 1] 抓取正文 → raw_material/*.md
    ↓
[Step 2] 提取个股结构化数据 → data/stocks_master_YYYY-MM-DD.json
    ↓
[Step 2.5] merge_new_stocks.py（合并到主数据）
    ↓
data/stocks/stocks_master.json（前端读取）
    ↓
[Step 3] 同步到 Firebase（可选）
    ↓
[Step 4] Git push → Vercel 自动部署
```

### 关键说明

- **Step 2.5 是关键步骤**：必须运行 `merge_new_stocks.py`，将中间产物合并到 `data/stocks/stocks_master.json`
- **旧版 `incremental_update.py`** 写入 skill 内部目录（`.trae/skills/.../data/master/`），前端不读取
- **行业/概念映射**：通过 `process_article.py` 自动从 `archived/同花顺行业.xls` 和 `archived/所属概念.xls` 获取

## ⚠️ 注意事项

1. **数组字段统一使用 `[]`**，不使用 `null`
2. **accidents 字段需优化**，超过 60 字简化
3. **source 链接修复**，`\_` → `_`
4. **title 字段必填**，格式统一为 `今天的一些信息整理MM.DD`
5. **industry 使用三级路径**，无空格，如 `"电子-电子化学品-电子化学品Ⅲ"`
6. **concepts 使用数组**，便于查询和过滤
7. **顶层是对象（keyed by code）**，不是数组
8. **必须运行 `merge_new_stocks.py`** 才能让前端看到新数据
