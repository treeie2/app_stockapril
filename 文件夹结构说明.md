## 📊 核心文件说明

### 数据文件（data/）

| 文件 | 说明 | 大小 | 更新频率 |
|------|------|------|----------|
| `stocks_master.json` | 完整股票数据（3157 只） | ~50MB | 每日 |
| `stocks_master.min.json` | 精简版（基本信息） | ~5MB | 每日 |
| `stocks_index.json` | 股票索引（快速查找） | ~1MB | 每日 |
| `groups.json` | 股票分组（锂资源、铝业等） | ~1KB | 按需 |
| `hot_topics.json` | 热点主题（算电协同、超节点） | ~50KB | 按需 |

### 脚本文件（.trae/skills/）

| 脚本 | 功能 |
|------|------|
| `wechat-fetch-research-embedded/` | 微信文章→原始素材→结构化数据 |
| `add-hot-topic/` | 管理热点和分组（添加/删除/列表） |

### Web 模板（templates/）

| 模板 | 用途 |
|------|------|
| `dashboard.html` | 主页仪表板 |
| `stocks.html` | 股票列表页 |
| `stock_detail.html` | 个股详情页 |
| `group_detail.html` | 分组详情页 |
| `hot_topic_detail.html` | 热点详情页 |

## 🗑️ 临时文件（不提交到 Git）

以下文件夹包含浏览器缓存等临时文件，已通过 `.gitignore` 排除：

- `.browser_profile/` - 浏览器配置文件
- `raw_material/.browser_profile/` - 临时浏览器缓存
- `*.html` (临时下载) - 临时 HTML 文件
- `ourcesappoutvsworkbenchcontribterminalcommonscriptsshellIntegration.ps1` - 终端临时文件

## 📝 文件整理规则

### ✅ 保留的文件

1. **核心数据**: `data/master/` 下的所有 JSON 文件
2. **脚本**: `.trae/skills/` 下的技能脚本
3. **模板**: `templates/` 下的 HTML 文件
4. **配置**: `.trae/rules/`, `vercel.json`, `.gitignore`
5. **文档**: `README.md`, `数据结构规范_v2.md`

### ❌ 删除的文件

1. **浏览器缓存**: `.browser_profile/` 所有内容
2. **临时文件**: `*.html` (临时下载), `tmp_*.txt`
3. **历史报告**: `archived/docs/` 下的部署报告
4. **旧脚本**: 已废弃的处理脚本
5. **重复数据**: 旧的 `stocks_master_*.json` 备份

## 🔄 数据流转

```
微信公众号文章
    ↓ (wechat-fetch-research-embedded)
raw_material/raw_material_YYYY-MM-DD.md
    ↓ (extract_stocks_from_raw_material)
data/master/stocks/2026-05-09.json
    ↓ (incremental_update)
data/master/stocks_master.json
    ↓ (sync_to_firestore)
Firebase Firestore
    ↓ (Vercel 部署)
Web 界面展示
```
