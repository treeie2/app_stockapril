## 🏗️ 整体架构

本系统采用**三层数据结构**：

```
┌─────────────────────────────────────┐
│     热点主题 (Hot Topics)           │  ← 宏观层面：市场热点、产业链
│  - 算电协同、超节点、固态电池...    │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│     股票分组 (Groups)               │  ← 中观层面：行业、概念、主题
│  - 锂资源、铝业、稀土...            │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│     个股研究 (Stocks)               │  ← 微观层面：具体公司
│  - 000001 平安银行、600519 贵州茅台... │
└─────────────────────────────────────┘
```

## 📈 个股数据

### 数据结构（v2.3）

采用**股票 - 文章两层结构**：

```json
{
  "stocks": {
    "688250": {
      // 第一层：股票基本信息
      "name": "盛科通信",
      "code": "688250",
      "board": "SH",
      "industry": "电子 - 半导体 - 集成电路",
      "concepts": ["交换芯片", "Scale-out", "Scale-up", "国产芯片"],
      "products": ["交换芯片"],
      "core_business": ["交换芯片研发与销售"],
      "industry_position": ["国产交换芯片龙头"],
      "chain": ["中游 - 芯片设计"],
      "partners": ["阿里巴巴", "字节跳动", "腾讯"],
      "mention_count": 2,
      "last_updated": "2026-05-09",
      
      // 第二层：文章详情
      "articles": [
        {
          "title": "今天的一些信息整理 5.7",
          "date": "2026-05-09",
          "source": "https://mp.weixin.qq.com/s/jU3eFc1irDolyJyLd6egyg",
          "accidents": ["二季度订单加速放量", "阿里 3 万 +scale out 订单"],
          "insights": ["MRC 协议影响有限，珍惜倒车接人机会"],
          "key_metrics": ["2026 年国产 GPU 出货量预计 400 万颗"],
          "target_valuation": ["约 2700 亿市值"]
        }
      ]
    }
  }
}
```

### 字段说明

#### 股票层级字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `name` | string | 股票名称 | "盛科通信" |
| `code` | string | 股票代码 | "688250" |
| `board` | string | 交易所板块 | "SH"（上海）、"SZ"（深圳） |
| `industry` | string | 所属行业 | "电子 - 半导体 - 集成电路" |
| `concepts` | array | 所属概念 | ["交换芯片", "国产芯片"] |
| `products` | array | 主要产品 | ["交换芯片"] |
| `core_business` | array | 核心业务 | ["交换芯片研发与销售"] |
| `industry_position` | array | 行业地位 | ["国产交换芯片龙头"] |
| `chain` | array | 产业链位置 | ["中游 - 芯片设计"] |
| `partners` | array | 合作伙伴 | ["阿里巴巴", "字节跳动"] |
| `mention_count` | integer | 文章数量 | 2 |
| `valuation` | object | 估值信息 | 见下方 |
| `last_updated` | date | 最后更新日期 | "2026-05-09" |

#### 文章层级字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `title` | string | 文章标题 | "今天的一些信息整理 5.7" |
| `date` | date | 文章日期 | "2026-05-09" |
| `source` | url | 来源链接 | 微信公众号链接 |
| `accidents` | array | 事件/催化剂 | ["订单加速放量"] |
| `insights` | array | 核心观点 | ["珍惜倒车接人机会"] |
| `key_metrics` | array | 关键数据 | ["出货量 400 万颗"] |
| `target_valuation` | array | 目标估值 | ["约 2700 亿市值"] |

### 排序规范 ⭐️

**所有数据必须按时间降序排序**（最新的在前）：

```python
# 股票列表按 last_updated 降序
stocks.sort(key=lambda x: x.get('last_updated', ''), reverse=True)

# 文章列表按 date 降序
for stock in stocks:
    if 'articles' in stock:
        stock['articles'].sort(key=lambda x: x.get('date', ''), reverse=True)
```

### 文件说明

1. **stocks_master.json**: 完整数据（包含所有字段）
2. **stocks_master.min.json**: 精简版（仅基本信息，用于快速加载）
3. **stocks_index.json**: 索引文件（快速查找股票代码）

1. **行业分类**: 锂资源、铝业、稀土、半导体...

3. **产业链**: 上游资源、中游制造、下游应用...
4. **自定义**: 根据研究需要自由组合

## 🌐 API 接口

### 个股 API

```
GET /api/stocks?code=688250
  获取指定股票详情

GET /api/stocks/search?q=盛科
  搜索股票

GET /api/stocks/all
  获取所有股票列表（精简版）
```

### 分组 API

```
GET /api/groups
  获取所有分组

GET /api/groups/{id}
  获取指定分组详情

POST /api/groups
  创建新分组

DELETE /api/groups/{id}
  删除分组
```

### 热点 API

```
GET /api/hot-topics
  获取所有热点

GET /api/hot-topics/{id}
  获取指定热点详情

POST /api/hot-topics
  创建新热点

DELETE /api/hot-topics/{id}
  删除热点
```

# 查找"盐湖股份"所属分组
stock_name = "盐湖股份"
for group in groups:
    if stock_name in group['stocks']:
        print(f"{stock_name} 属于分组：{group['name']} ({group['description']})")

```python

# 加载热点数据

        print(f"相关股票：{', '.join(topic['stocks'])}")
```

## 🆘 常见问题

### Q1: 如何处理重复股票代码？
A: 使用 `check_and_merge_duplicates.py` 检查，然后运行对应的合并脚本。

### Q2: 如何添加新的热点主题？
A: 使用 `add-hot-topic` 技能或手动编辑 `data/hot_topics.json`。

### Q3: 如何同步到 Firebase？
A: 运行 `sync_stocks_to_firebase.py` 和 `firebase_hot_topics.py`。

### Q4: 数据文件太大怎么办？
A: 使用精简版 `stocks_master.min.json` 或按日期分片存储。
